따라서 저는 이 UI를 단순한 목업이 아닌, 재사용 가능한 **‘법률 보고서 디자인 시스템 컴포넌트 키트’** 형태로 접근하겠습니다.

### 🔍 리스크 보고서 (Report View) 디자인 전략: 권위와 위계 구조 구축

#### 1. 핵심 컨셉 및 목표
*   **컨셉:** 법무법인/국가 공공기관에서 발급하는 전자문서(PDF 기반)의 인터페이스 재현.
*   **목표:** 리스크 점수와 진단 코드를 최상위 정보로 배치하여, 사용자가 보고서를 읽는 즉시 "이것을 모르면 큰일 난다"라는 느낌을 받게 한다.
*   **톤앤매너:** 엄격함(Rigorous), 객관적임(Objective), 권위적임(Authoritative).

#### 2. 디자인 시스템 컴포넌트 정의 (Figma/Sketch 기반 구조)

저는 다음 세 가지 핵심 컴포넌트로 보고서 페이지를 분할하고, 이들이 결합하여 하나의 통합된 경험을 만듭니다.

**A. [헤더 및 법적 고지 영역] (The Authority Block)**
*   **목적:** 누가(Who), 무엇을(What), 언제(When) 분석했는지 명시하여 신뢰도를 극대화합니다.
*   **요소:** 공식 로고 배치, 문서 제목(`법률 리스크 진단 보고서`), 버전/일자 워터마크, 고지 사항 (예: "본 보고서는 참고 자료이며, 최종 법적 효력은 아닙니다.").

**B. [핵심 요약 및 위험 평가 모듈] (The Risk Dashboard)**
*   **목적:** 페이지에 들어온 순간 가장 먼저 눈에 띄는 곳. 데이터 분석의 결과를 즉각적으로 제시합니다. **(최우선 구현 영역)**
*   **요소:**
    1.  **Diagnosis Code & Title:** 가장 크게, 중앙 상단에 위치 (예: `[Code L-403]: 재개발 인허가 절차상 중대한 법적 리스크 존재`). 폰트는 명조체나 세리프 계열의 권위적인 느낌을 사용합니다.
    2.  **Risk Score Visualization:** 단순 숫자가 아닌, 게이지(Gauge) 또는 색상 블록으로 시각화합니다. (예: 위험도 레벨 [Critical/High/Moderate])
    3.  **Summary Impact Statement:** 리스크 점수와 진단 코드를 근거로 한 한 문장 요약 결론 ("즉각적인 전문 자문이 필요하며, 추가 피해액 발생 가능성이 높습니다.").

**C. [상세 분석 및 데이터 출처 섹션] (The Credibility Deep Dive)**
*   **목적:** 보고서의 깊이를 증명하고, 각 주장이 '어떤 데이터'에 근거했는지 추적할 수 있게 합니다.
*   **요소:**
    1.  **섹션 분리:** `[네이버 매물 정보]`, `[법제처 법규 분석]`, `[정보몽땅 진행 상황]` 등 출처별로 명확히 구분합니다 (얇은 그리드 라인 또는 섹션 타이틀 사용).
    2.  **데이터 출처 표시:** 각 수치 옆에 작은 아이콘과 함께 데이터 출처를 표기합니다. (예: 📍 법제처 API 기반 / 📊 네이버 크롤링 V2.1) — 이는 *투명성*을 높여 신뢰도를 만듭니다.
    3.  **상충 위험 분석:** 여러 소스가 충돌하는 경우, 그 상충점을 명확히 박스 처리하고 해석을 제공합니다 (예: ⚠️ [Conflict Alert] 매매가와 법규 기반 시세 간의 괴리율이 높음).

#### 3. 디자인 실행 (구체적 산출물)

이러한 구조를 바탕으로, 가상의 '법률 리스크 보고서' 목업을 구체적인 Figma 컴포넌트 지침과 함께 작성하겠습니다.

## 📑 컴포넌트 상세 스펙 및 배치 지침

### 1. [Report Header] - 법적 권위 확립 영역
| 요소 | 내용/기능 | 디자인 가이드 |
| :--- | :--- | :--- |
| **제목** | 대한민국 재개발 리스크 진단 보고서 | 폰트: 세리프 계열 (Times New Roman 등), 크기: H1, 색상: Primary. |
| **워터마크** | [CONFIDENTIAL] / 법률 자문용 문서 | 배경 전체에 희미하게 배치 (Opacity 5%). 페이지의 권위와 기밀성을 부여. |
| **제작 주체** | 대한민국 재개발 No.1 플랫폼 (법무법인 OOO 협력) | 공식 로고 및 '진단 기관명'을 상단 좌/우에 대칭 배치. |

### 2. [Risk Summary Block] - 가장 중요한 정보 (최상위 가시성 확보)
*   **배치:** 보고서의 페이지 최상단, 메인 콘텐츠 시작 지점에 고정 배치.
*   **A. Diagnosis Code Card (코드 박스):**
    *   **디자인:** 배경을 밝은 그레이 (#F5F5F5)로 설정하고 테두리를 두껍게 처리합니다. 마치 '공식 결론'이 담긴 패널처럼 보이게 합니다.
    *   **내용:** `[L-403]` (Diagnosis Code). 폰트 크기: 가장 크게, 볼드체.
    *   **결론:** **"재개발 인허가 절차상 중대한 법적 리스크 존재"**. 이 문구가 코드를 설명하는 역할을 합니다.

*   **B. Risk Level Indicator:**
    *   **디자인:** 크고 직관적인 게이지 형태 또는 3단계 컬러 블록 (Critical/High/Moderate).
    *   **강조:** 위험도가 높을수록 배경색을 **Impact Red**로 채우고, 그 위에 "즉각적 조치 필요"라는 경고 문구를 배치합니다.

### 3. [Detailed Analysis Section] - 데이터 출처와 해석의 분리
*   **구분자 역할:** 각 섹션은 좌측 또는 우측에 **얇지만 명확한 수직선 그리드(Grid Line)**를 사용하여 구분합니다. (법률 문서의 구획 느낌)
*   **데이터 조합 시각화:** 모든 데이터는 '원본 정보'와 '플랫폼 해석/리뷰어 코멘트'가 분리되어 제시됩니다.

| 소스 영역 | 원본 데이터 표시 (`Raw Data`) | 플랫폼 분석 레이어 (`Platform Analysis`) |
| :--- | :--- | :--- |
| **네이버 매물** | (매물 가격, 면적) - 출처: Naver API V2.1 🌐 | [분석] *해당 지역의 최근 유사 사례 대비 평균가보다 15% 높음.* (위험도 판단 근거 제시) |
| **법제처 법규** | (재개발 절차법 제XX조) - 출처: Law API v3.0 📜 | [리스크] *현 매물은 공사비 상승률을 반영하지 못하여, 향후 민원 발생 시 법적 분쟁 리스크가 높음.* (전문 해석 제시) |
| **정보몽땅** | (구역 지정 현황: 초기 단계) - 출처: Info API v1.5 🗺️ | [권고] *초기 단계는 변수가 가장 많으므로, 공인중개사 협회 전문 자문을 통해 리스크를 최소화해야 함.* (CTA로 연결되는 핵심 메시지) |
