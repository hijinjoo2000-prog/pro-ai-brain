## 💻 [Mockup Design Brief] 유료 결제 유도 페이지 메인 섹션
(Target Device: Desktop, Professional View)

### 🎨 디자인 시스템 (Design System Application)
*   **주요 색상 (Primary):** Dark Navy (#102A43) - 신뢰성, 전문성.
*   **보조 색상 (Secondary):** Slate Gray (#C9D3DD) - 배경, 정보 분리.
*   **경고색/강조색 (Accent):** Impact Red (#D83F40) - 위험(Risk), 결제 유도 포인트.
*   **타이포그래피:** 고딕 계열의 세리프가 가미된 폰트 사용 (예: Noto Sans KR Bold / Georgia 유사체).

### 🖼️ 섹션별 레이아웃 및 컨셉 (Section Breakdown)

#### 🔴 SECTION 1: 헤드라인 및 후킹 (The Hook - Top Fold)
*   **목표:** 공인중개사의 불안감(Liability Gap)을 건드리며, 현 시스템의 한계를 지적한다.
*   **레이아웃:** 대형 타이포그래피와 대비되는 여백 활용.

| 요소 | 내용 및 디자인 가이드 | 의도/효과 |
| :--- | :--- | :--- |
| **메인 제목 (H1)** | **[재개발 매물, 단순 시세가 아닙니다.]** <br> *데이터로 증명하는 법률 리스크 진단 보고서.* | "시세"라는 일반적인 개념에서 벗어나 '법률적 관점'으로 프레임을 전환. 전문가적 영역에 침투. |
| **서브 카피 (P)** | 복잡하게 얽힌 개발 규정, 구역별 법제 변화, 그리고 네이버 매물 데이터의 논리적 충돌 지점을 AI가 자동으로 분석합니다. **직관적인 리스크 점수(Risk Score)로 한눈에 파악하세요.** | 우리 서비스의 핵심 가치(자동화된 다중 검증)를 명확히 제시. |
| **Call to Action (CTA)** | [✨ 지금, 법률 리스크 진단 보고서 받아보기] (Impact Red 버튼) | 구매가 아닌 '진단'이라는 용어로 심리적 장벽을 낮추면서도 강한 행동 유발. |

#### 🔵 SECTION 2: 문제 제기 및 비교 분석 (Problem vs. Solution)
*   **목표:** 기존 방식(경쟁사, 수동 검토)으로는 절대 파악할 수 없는 정보의 존재를 시각적으로 증명한다.
*   **레이아웃:** 좌우 3단 분할 레이아웃.

| 요소 | [기존 분석 (Manual)] | [우리 서비스 (Automated)] | 강조점 |
| :--- | :--- | :--- | :--- |
| **제목** | *❌ 직감과 경험에 의존하는 방식* | **✅ 데이터 기반의 논리적 검증 시스템** | 대비 효과 극대화. |
| **내용 1 (Icon)** | 법제처 API 수동 크롤링 및 교차 검증 (시간 소모) | 네이버 + 법제처 + 정보몽땅 통합 API 호출 (즉시 분석) | *속도* 강조. |
| **내용 2 (Impact Red Box)** | 충돌 지점 파악 어려움: '이건 좀 아닌데?'라는 느낌만 남음. | **[Conflict Details]**: 법적 모순 지점을 **근거 필드와 함께 정량화**하여 제시합니다. (API v3.0 활용) | *전문성* 강조. 가장 핵심적인 판매 포인트. |
| **내용 3 (Icon)** | 최종 리스크 점수: 주관적인 판단에 의존하는 단일 점수. | **[Risk Score]**: 데이터 충돌 및 법률 해석의 누적 가중치를 반영한 객관적 스코어(0.0~1.0). | *객관성* 강조. |

#### 🟢 SECTION 3: 핵심 시각화 영역 (The Proof - Data Visualization)
*   **목표:** API v3.0의 가장 복잡하고 가치 있는 데이터(충돌 지점)를 아름다운 '보고서' 형태로 보여주어, 이것을 얻기 위해 돈을 내야 한다고 느끼게 만든다.
*   **레이아웃:** 전체 너비에 걸친 대형 카드 형태 (Card Style).

1.  **Report Header:** "재개발 법률 리스크 종합 진단 보고서 (Matter ID: M-2024XXXX)"
2.  **Summary Card:** 가장 눈에 띄게 배치.
    *   `최종 리스크 스코어`: **0.78 / 1.0** (Impact Red 배경, 큰 글자)
    *   `Conflict Count`: **3건의 법적 모순 감지** (Warning Icon과 함께 강조)
    *   `핵심 요약 보고서`: "해당 매물은 [구역 A]와 [법률 B] 간에 중대한 충돌 위험을 내포하고 있습니다. 추가 컨설팅이 필수입니다."
3.  **Detailed Conflict Table (Critical Area):** (가장 깊이 있게 보여주어야 할 부분)

| 필드 이름 | 원본 데이터 출처 | 충돌 내용 / 모순 근거 | 영향도 | 해결 필요성 |
| :--- | :--- | :--- | :--- | :--- |
| **용적률 제한** | 법제처 API (최신 개정안) | 2024년 A법 상향 vs. 네이버 매물 정보 기준: *충돌 지점* | High (Impact Red) | **필수 검토** |
| **진행 구역 지정 여부** | 정보몽땅 API (구역 B) | 과거 자료(2021년)와 현재 공지 간의 시간적 불일치. | Medium | 권장 검토 |

*   **UX/Interaction:** 이 테이블 영역에 마우스를 올리면, "해당 충돌 지점을 해결하기 위한 **전문 컨설팅 가이드라인**이 제공됩니다."라는 팝업 메시지를 띄운다. (-> 유료 결제 연결 고리)

#### 💰 SECTION 4: 구매 전환점 및 가격 모델 제시 (The Paywall)
*   **목표:** 모든 것을 경험한 후, 이제는 '구매'만이 답임을 느끼게 한다.
*   **레이아웃:** 명확한 3단 박스형 가격 비교.

| 항목 | [기본 진단 리포트] | **[프로 컨설팅 패키지 - 추천]** | [API 연동 개발 라이선스] |
| :--- | :--- | :--- | :--- |
| **가격** | ₩39,000 (1회) | **₩150,000 (월 구독)** | 문의 |
| **제공 범위** | 리스크 스코어, 충돌 개수 보고서 제공. | **+ 상세 컨설팅 가이드라인 접근권**, 무제한 법률 비교, 전담 분석가 연결. | 시스템 API 연동 및 트래킹 기능. |
| **CTA 버튼** | [진단 받기] (Gray) | **[전문가 플랜으로 결제하기] (Impact Red)** | [문의하기] (Ghost Button) |
