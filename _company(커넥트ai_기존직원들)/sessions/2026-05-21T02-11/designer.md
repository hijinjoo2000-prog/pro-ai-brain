# 📄 [High-Fidelity Mockup] 플랫폼 컨설팅 서비스 랜딩 페이지 (Ver. Final)

## 🎨 디자인 가이드라인 요약
*   **톤앤매너:** 법무법인, 금융기관 리스크 보고서. 극도의 전문성(Authority)과 긴급함(Urgency).
*   **주 색상:** 네이비/차콜 그레이 (`#1A237E`, `#455A64`) - 신뢰성.
*   **강조 색상:** 임팩트 레드 (Impact Red, `#D32F2F`) - 위험 경고, 긴급성.
*   **폰트:** 고딕 계열의 명조 느낌이 가미된 폰트 (예: Pretendard Bold/SemiBold).

## 🚀 SECTION 1. HERO / 문제 제기 (The Hook: 지금 무엇을 놓치고 있습니까?)

(최상단, 페이지 로딩 시 즉시 보이는 영역)

**[배경]:** 어둡고 차분한 네이비 배경에 데이터 그리드 패턴을 은은하게 깔아 '데이터 기반 분석' 느낌 강조.
**[헤드라인 (H1)]: 법적 리스크가 당신의 수익률을 갉아먹고 있습니다.**
> *부동산 정보 검색만으로는 결코 알 수 없는, 잠재적 손실(Loss)에 대한 정량적 보고서.*

**[서브 카피]:** 대한민국 재개발 시장에서 발생하는 법규 충돌 및 행정 절차상의 사각지대를 AI가 분석하여 '손실 회피 비용'으로 제시합니다.
*   **핵심 기능 요약 (3가지 아이콘):**
    1.  🔍 **법제처 연동:** 최신 재개발/재건축 법규 자동 반영.
    2.  📊 **충돌 검증 엔진:** 크롤링 데이터와 법적 규정 간의 논리적 모순을 점수화.
    3.  🛡️ **리스크 예측 보고서:** 매물별 위험도를 '점수'로 제시하여 의사결정을 지원.

**[Call to Action (CTA 1 - 낮은 진입 장벽)]:**
> $\rightarrow$ **지금, 무료 리스크 점수 사전진단 받기 (API 기반)** *(버튼: 차분한 네이비)*

## 🚨 SECTION 2. 불안 증폭 / 문제 심화 (The Pain Point: 당신의 정보는 불완전합니다.)

(사용자가 스크롤을 내릴수록 긴장감이 고조되는 섹션)

**[제목]: 법적 모순은 눈에 보이지 않습니다. 하지만 가장 위험합니다.**
*   "공인중개사님의 경험과 네이버 매물 데이터는 '사실'만을 알려줄 뿐입니다. 저희 플랫폼은 그 사실들 사이의 '법률적 공백(Gap)'을 찾아냅니다."

### 🔍 Case Study 시뮬레이션: [Impact Red Box]
*(이 부분이 핵심입니다. 일반적인 정보 제공 페이지와 완전히 분리된, 법률 문건처럼 보이도록 디자인합니다.)*

**[가상 매물 분석 보고서 (좌측)]:**
> **매물명:** OOO 아파트 재개발 구역 A-3 블록
> **기초 데이터 요약:** (네이버/법제처 API 연동 정보 나열)
> *   건축연도: 1985년 / 용적률: 200% / 법정 주거지역 등...

**[분석 결과 및 리스크 엔진 보고서 (우측 - Impact Red Box)]:**
*(전체적으로 회색/네이비 바탕에, 이 박스만 빨간 경고 라인으로 둘러싸임)*

> **🔴 [CRITICAL RISK ALERT] 법률적 충돌 발생 가능성 감지 (Risk Score: 85점 / 심각)**
> *   **진단 근거:** 현장 사진 기반의 구조물(A)은 해당 구역 지정 당시의 건축법 기준(B)과 논리적으로 모순됩니다. 특히, **[정보몽땅 API]**가 제시하는 A-3 블록 주변 2차 개발 계획(C)과의 법적 간섭 가능성(D)이 매우 높습니다.
> *   **결론:** 단순 현장 확인으로는 파악할 수 없는 '절차적 리스크'를 내포하고 있습니다. **추가적인 전문가 분석 없이는 계약 진행 시 막대한 손실을 입을 위험이 존재합니다.**

## ⚙️ SECTION 3. 독점적 가치 증명 (The Solution: 오직 이 엔진만이 가능합니다.)

(왜 다른 곳에서는 안 되는지 설명하는 섹션)

**[제목]: 법률 리스크 점수 산출 과정 공개: 플랫폼의 기술적 우위.**
*   **내용:** 단순한 정보 비교가 아님을 강조하며, 데이터 흐름도를 시각적으로 보여줍니다.

**[Flow Diagram (간소화된 API 연결도)]:**
1.  **Input Layer:** 네이버 매물(현장 데이터) $\rightarrow$ 법제처 API(공적 규정) $\rightarrow$ 정보몽땅 API(구역 변동 이력)
2.  **Processing Core:** **[독점 알고리즘]** (데이터 타입 불일치, 논리적 모순 감지 → 가중치 부여)
3.  **Output Layer:** **Risk Score (0~100)** $\rightarrow$ 상세 리스크 보고서 출력

> *이처럼 여러 데이터셋의 충돌 지점을 통합적으로 해석하는 것은 고도의 법률 분석 엔진 없이는 불가능합니다.*
