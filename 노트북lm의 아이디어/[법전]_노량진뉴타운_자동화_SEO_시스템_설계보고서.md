## 3. 통합 자동화 시스템 아키텍처 (4개 레이어)

### 3.1 GUI 및 스레드 제어 레이어 (PyQt6)
- QLineEdit, QComboBox로 구역 번호, 프리미엄, 초기 투자금 입력
- QThread 기반 백그라운드 워커로 UI 프리징 방지

### 3.2 이미지 부품 생산 레이어 (Batch Automation)
- `구역정보만들기.bat` → 구역 거시 데이터 인포그래픽
- `매물카드생성기.bat` → property_maker_final_v61.py 호출, 핵심 정보 이미지
- `주소카드생성기.bat` → adress.py 실행, 위치 지도 이미지
- `썸네일만들기.bat` → 정방형(1:1 비율) 대표 이미지

### 3.3 지능형 상위 노출 사전 검증 모듈 (NLP Validation Engine)

```
1단계: 실시간 경쟁사 문서 크롤링
└─ Selenium + BeautifulSoup으로 네이버 VIEW 탭 상위 1~3위 URL 수집
└─ 각 페이지 본문 텍스트 스크래핑 (글자 수, 이미지 개수)

2단계: 형태소 분석 기반 LSI 키워드 추출
└─ KoNLPy Okt 엔진으로 명사형 형태소 추출
└─ 상위 15개 키워드 풀 도출 (조합원, 입주권, 이주비, 분담금, 프리미엄 등)

3단계: AI 원고 생성 및 스코어링

검증 기준 (3가지 모두 통과 필요):
① 생성된 글자 수 ≥ 상위 3개 문서 평균 글자 수의 85%
② 추출된 상위 필수 형태소 중 최소 70% 이상 본문 포함
③ 메인 키워드 밀도 5% 미만 (스팸 분류 방지)

→ 실패 시 최대 3회까지 자동 재작성
```

### 3.4 RPA 자동 포스팅 레이어 (Selenium Automation)
- **봇 탐지 우회**: pyperclip을 이용한 클립보드 복사-붙여넣기
- **human_typing 함수**: 사람의 타이핑 리듬 모사 딜레이 알고리즘
- **포스팅 경로**: `https://blog.naver.com/{ID}?Redirect=Write`
- **CTA 삽입**: 카카오톡 오픈채팅 링크 + 오프라인 명함 이미지 자동 첨부

## 5. 실전 투자 법률 & 절세 핵심

### 다주택자 취득세 절세 (토지/멸실 상태 매수)
- 관리처분인가 후 철거 완료 → 토지(나대지) 상태
- **토지 취득세율 4.6% 단일 적용** (다주택자 중과세 회피)
- vs 기축 아파트 추가 취득 시 최대 12%

### 이주비 대출 승계 전략
- 원조합원이 받은 이주비 대출을 매수자가 합법적으로 승계
- 감정평가액의 50~60% 선 HUG 보증, 저금리 적용
- 초기 투자금 절감 효과 → 레버리지 극대화

### 일시적 2주택 비과세 출구 전략
- 입주권은 **취득세**: 토지(4.6%) / **양도세**: 주택 수에 포함
- 기존 주택 보유 시, 준공 후 '종전 주택 처분 기한' 내 매도 필수
- 기한 초과 시 → 비과세 불가 + 중과세 적용
