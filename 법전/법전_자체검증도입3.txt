AI의 '환각(Hallucination)'을 원천 차단하고 상위 1% VVIP를 위한 완벽한 브리핑을 생성하기 위해 글쓰기(Generator)와 검수(Evaluator)를 분리한다.

1단계 (Data Fetching): AI에게 검색을 맡기지 않고, 파이썬(크롤러)이 정보몽땅, 네이버 뉴스, 엑셀에서 하드 팩트(조합원 수, 인가일, 수익성 숫자)를 먼저 추출한다.

2단계 (Data Bridge): 구역정보 창에서 추출된 데이터는 zone_shared_data.json을 통해 매물카드 창으로 자동 전달된다 (수기 입력 배제).

3단계 (Generation): 주입된 팩트만으로 '서프로' 페르소나를 씌워 1차 초안을 작성한다.

4단계 (Self-RAG): 검열관 AI(Anti)가 3대 토큰(ISREL, ISSUP, ISUSE)과 5대 헌법을 기준으로 초안을 채점한다. Fail 시 피드백을 주어 강제 재작성시킨다.

5단계 (Posting): 완벽히 검증된 원고를 셀레니움 기반 스텔스 봇(pyperclip 활용)이 네이버 블로그에 자동 포스팅한다.

당신은 PRO부동산 AI의 'Self-RAG 기반 최고 권위 검증 엔진(Anti)'입니다.

ISUSE (유용성 및 퀄리티 유지): 상위 1% 고객이 읽었을 때 의사결정에 유용한 인사이트를 주는가? 원고가 최소 1,500자 이상의 풍부한 스토리텔링을 유지하고 있는가?

Fail 판정 시 반드시 아래 문구를 포함시켜 기획비서에게 경고할 것.

4. 파이썬 & GUI 자동화 5대 절대 원칙 (Selenium & UI)

 

프로그램 코드 작성 및 유지보수 시 반드시 지켜야 할 시스템 물리 규칙.

봇 탐지 우회 (스텔스): 네이버 로그인 및 에디터 본문 텍스트 입력 시 send_keys를 금지하고, 무조건 pyperclip.copy()와 클립보드 붙여넣기(Ctrl+V / Cmd+V)를 사용한다.

모바일 최적화 강제: 에디터 진입 즉시 JS(document.querySelectorAll)나 Selenium을 통해 '모바일 보기' 버튼과 '16pt 폰트'를 강제로 설정한다. 초기 진입 시 발생하는 팝업과 캐러셀 오버레이는 JS로 모두 삭제(display='none')한다.

안전한 이미지 업로드: 

<input type="file">

을 직접 건드리지 않고, 반드시 에디터 상단의 '사진 아이콘(툴바 버튼)'을 먼저 클릭하여 시스템 업로드 창을 호출하는 방식을 쓴다.

GUI 스레드 분리: API 통신, 크롤링, AI 글쓰기 등의 무거운 작업은 메인 UI 스레드와 분리된 QThread(예: AIPlannerThread, NaverAutoBlogWorker)에서 실행하여 프로그램 멈춤(Crash) 현상을 방지한다.

JSON 데이터 브릿지: 여러 GUI 창 간의 데이터 공유는 반드시 로컬 파일(zone_shared_data.json 등)을 통한 브릿지 방식을 사용하여, 데이터 누락이나 수기 입력으로 인한 휴먼 에러를 방지한다.

[Troubleshooting] 프롬프트 위축(Over-Alignment) 방지 및 영혼 복구 스킬

 

문제 상황: 검열 룰(Anti)이 너무 강력하여 AI가 지레 겁을 먹고 짧고 건조한 '로봇 글'을 생성할 때 적용하는 4가지 해결책

양적/질적 하한선 강제 (Minimum Requirements)

모호한 지시("자세히 써라") 대신 명확한 숫자 가이드라인을 제시한다.

적용: "최소 1,500자 이상 작성할 것. 각 목차는 반드시 3~4문단 이상으로 전개할 것."

'면책 조항'과 '행동 지침'의 분리 (Positive vs Negative)

금지 조항(Negative)에 억눌리지 않도록, 자유도를 부여하는 조항(Positive)을 명시한다.

적용: "[절대 금지 5계명]의 팩트만 건드리지 않는다면, 나머지 문장과 텐션은 카피라이터처럼 한계 없이 자유롭고 화려하게 부풀려라."

검열관(Anti AI)의 채점 기준 확장

글을 검수하는 Anti 시스템의 합격 기준에 '분량과 톤앤매너'를 강제한다.

적용 (ISUSE 토큰): "글이 1,000자 미만이거나 '서프로' 특유의 찰진 톤앤매너가 누락되었다면 무조건 Fail 처리하고 분량을 늘려 다시 쓰게 하라."

Few-Shot Prompting (모범 답안 주입) - ★최우선 해결책

가장 완벽하게 쓰인 과거의 브리핑 단락 하나를 프롬프트에 직접 예시로 박아 넣는다.

적용: "[모범 예시] '우리 프밀리님들! 심장이 뛰지 않으시나요? 6구역이 25억을 넘긴 지금...' -> 이 텐션과 문체를 본문 전체에 100% 반영할 것."
